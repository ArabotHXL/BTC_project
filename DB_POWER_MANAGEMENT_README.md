# 基于数据库的智能电力削减管理系统

## 系统概述

智能电力削减管理系统是一个基于数据库的高级工具，用于帮助比特币矿场根据矿机健康状态和效率智能管理电力削减策略。系统通过持久化存储和高级分析功能，提供更全面的矿场管理能力。

## 核心功能

1. **矿机状态持久化管理**：所有矿机状态信息存储在数据库中，确保数据持久性和一致性
2. **智能电力削减策略**：基于矿机健康分数和效率，自动选择最佳的关停顺序
3. **矿机轮换计划**：自动创建和执行矿机轮换计划，延长矿机寿命
4. **数据可视化**：通过图表和仪表盘直观展示系统状态和历史趋势
5. **操作记录与审计**：记录所有矿机操作，支持后续分析和审计
6. **性能快照**：定期创建系统性能快照，用于长期趋势分析

## 技术架构

- **后端**：Flask Web框架
- **数据库**：PostgreSQL关系型数据库
- **ORM**：SQLAlchemy对象关系映射
- **前端**：Bootstrap + 原生JavaScript
- **图表**：Chart.js数据可视化库

## 系统结构

本系统由以下核心组件构成：

1. **数据库模型** (`power_management_models.py`)：
   - 矿机状态表
   - 操作记录表
   - 电力削减计划表
   - 轮换计划表
   - 性能快照表

2. **数据库访问层** (`power_management_db.py`)：
   - 提供数据库CRUD操作的封装
   - 处理数据库事务和异常

3. **业务逻辑层** (`db_power_manager.py`)：
   - 实现核心业务逻辑
   - 处理电力削减策略计算
   - 管理矿机轮换

4. **Web应用层** (`db_power_management_app.py`)：
   - 提供Web界面和API
   - 处理用户请求和响应

5. **前端界面** (`templates/db_power_dashboard.html`)：
   - 直观的仪表盘展示
   - 矿机管理界面
   - 电力控制界面
   - 历史数据图表

## 快速开始

### 前提条件

- Python 3.8+
- PostgreSQL数据库
- 建议使用虚拟环境

### 设置数据库

系统需要数据库连接。可以通过环境变量`DATABASE_URL`设置数据库连接串。例如：

```
export DATABASE_URL="postgresql://username:password@localhost:5432/power_management"
```

如果未提供数据库URL，系统将使用SQLite数据库作为默认值。

### 运行系统

```bash
python run_db_power_manager.py
```

首次运行时，系统会：
1. 创建必要的数据库表
2. 初始化1000台模拟矿机数据用于测试
3. 启动Web服务器

### 访问管理界面

打开浏览器访问：http://localhost:5002/

## 使用指南

### 系统仪表盘

仪表盘提供系统整体状态的直观视图：
- 总矿机数、运行中矿机数、总算力、电力削减比例
- 各类别矿机状态统计
- 最近操作记录

### 矿机管理

矿机管理页面允许您：
- 查看所有矿机的详细信息
- 按状态和类别筛选矿机
- 手动控制矿机的启动和关停
- 查看矿机详情和操作历史

### 电力控制

电力控制页面提供：
- 设置电力削减目标百分比
- 配置关停优先级顺序
- 应用电力削减策略
- 生成和执行矿机轮换计划

### 历史数据

历史数据页面展示：
- 系统性能趋势图表
- 历史性能数据表格
- 可配置的时间范围

### 操作记录

操作记录页面提供：
- 所有矿机操作的详细记录
- 操作类型、时间、操作人和原因
- 操作结果状态

### 计划管理

计划管理页面用于：
- 查看所有电力削减计划
- 激活/停用计划
- 管理轮换计划

## 高级功能

### 智能关停策略

系统会基于以下因素智能选择要关停的矿机：
1. 矿机健康状态类别（D>C>B>A）
2. 矿机效率（低效率优先关停）
3. 用户配置的优先级顺序

### 矿机轮换

为了延长矿机使用寿命，系统支持轮换功能：
1. 自动选择健康分数最低的运行中矿机关停
2. 启动之前关停的矿机
3. 按用户设定的时间间隔执行轮换

### 性能快照

系统支持创建性能快照，用于长期趋势分析：
1. 记录每日系统状态
2. 提供历史趋势图表
3. 支持不同时间范围的数据分析

## 系统维护

### 创建性能快照

可通过以下方式创建性能快照：
1. 在系统工具栏中点击"创建性能快照"按钮
2. 系统会自动捕获当前系统状态并存储到数据库

### 重新初始化矿机数据

对于测试环境，可以重新初始化矿机数据：
1. 在系统工具中输入矿机数量
2. 点击"初始化"按钮
3. 系统将重新生成模拟矿机数据
