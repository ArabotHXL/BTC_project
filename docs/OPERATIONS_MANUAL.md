## 1. AI 自动执行系统操作指南

### 1.1 什么是 AI 自动执行

AI 自动执行系统是一个智能运维助手，能够：
- **自动检测**：实时监控矿机状态、温度、算力等指标
- **智能诊断**：分析告警根因，提供诊断报告
- **生成建议**：根据诊断结果生成操作建议
- **自动/半自动执行**：低风险操作自动执行，高风险操作需人工审批

**完整闭环流程：**
```
告警(Detect) → 诊断(Diagnose) → 建议(Recommend) → 审批(Approve) → 执行(Act) → 审计(Audit) → 复盘(Learn)
```

### 1.2 风险等级说明

系统将所有操作分为四个风险等级：

| 风险等级 | 英文标识 | 说明 | 示例操作 | 审批要求 |
|---------|---------|------|---------|---------|
| 🟢 低风险 | LOW | 影响范围小，可逆性高 | 调整风扇速度、单台矿机重启 | 可自动执行 |
| 🟡 中风险 | MEDIUM | 影响适中，需关注 | 多台矿机重启、切换矿池 | 操作员审批 |
| 🟠 高风险 | HIGH | 影响较大，需谨慎 | 批量关机、限电操作 | 管理员审批 |
| 🔴 严重风险 | CRITICAL | 影响重大，不可逆 | 大规模关机、系统配置变更 | 管理员审批 + 二次确认 |

**风险评估因素：**
- 操作类型（关机风险 > 重启 > 调参）
- 影响设备数量（设备越多风险越高）
- 功率影响（kW）
- 收益影响百分比
- AI 置信度

### 1.3 如何启用/禁用自动执行

#### 启用自动执行

1. 进入 **托管管理** → **自动化规则**
2. 点击 **新建规则** 按钮
3. 配置自动审批条件：

│  站点:     [选择矿场                    ▼]        │

1. 进入 **托管管理** → **自动化规则**
2. 找到目标规则
3. 点击规则右侧的 **禁用** 开关
4. 确认禁用

> ⚠️ **注意**：禁用自动执行后，所有操作都需要人工审批。建议在系统维护或异常期间临时禁用。

## 2. 审批流程

### 2.1 自动审批规则 (AutoApproveRule)

自动审批规则允许系统在满足条件时自动批准 AI 建议，无需人工干预。

**规则配置项：**

| 配置项 | 说明 | 示例值 |
|-------|------|--------|
| 规则名称 | 规则的描述性名称 | "低风险自动批准" |
| 适用站点 | 规则适用的矿场 | Site A, Site B |
| 允许风险等级 | 可自动批准的最高风险 | LOW |
| 允许操作类型 | 可自动执行的操作 | 重启、调整风扇 |
| 最大设备数 | 单次操作最多设备 | 5 |
| 生效时间窗口 | 规则生效时间段 | 08:00-20:00 |
| 最小置信度 | AI 置信度阈值 | 0.8 (80%) |

**创建自动审批规则步骤：**

1. 导航至 **设置** → **自动化规则**
2. 点击 **添加规则**
3. 填写规则配置
4. 点击 **保存**
5. 测试规则（可选）

### 2.2 手动审批流程

当 AI 建议需要人工审批时，按以下流程操作：

**步骤 1：查看建议详情**

1. 在待审批列表中点击 **查看详情**
2. 仔细阅读以下信息：
   - **触发原因**：什么事件触发了此建议
   - **AI 诊断**：系统分析的根本原因
   - **建议操作**：推荐执行的操作
   - **影响评估**：预计影响的设备和收益
   - **风险评估**：风险等级和评估因素

**步骤 2：做出审批决定**

- **批准**：点击 **批准执行** 按钮

  - 系统将立即开始执行操作
  
- **拒绝**：点击 **拒绝** 按钮
  - 必须：填写拒绝理由
  - 建议说明替代方案

**步骤 3：跟踪执行结果**

批准后，可在 **执行记录** 页面跟踪操作状态。

### 2.3 不同风险等级需要的审批级别

| 风险等级 | 所需审批角色 | 审批时限 | 备注 |
|---------|-------------|---------|------|
| LOW | 可自动审批 | 立即 | 需配置自动审批规则 |
| MEDIUM | Operator (操作员) | 2小时内 | 超时自动过期 |
| HIGH | Manager (管理员) | 1小时内 | 需管理员及以上权限 |
| CRITICAL | Admin (超级管理员) | 30分钟内 | 需二次确认 |

**权限层级：**
```
Operator (操作员) < Manager (管理员) < Admin (超级管理员)
```

> 💡 **提示**：更高权限的用户可以审批任何级别的建议。

### 2.4 拒绝建议时需要提供理由

拒绝 AI 建议时，**必须**提供拒绝理由。这有助于：
- 记录决策过程，满足审计要求
- 帮助 AI 系统学习优化
- 便于其他团队成员了解情况

**常见拒绝理由示例：**

| 场景 | 拒绝理由示例 |
|-----|-------------|
| 时机不当 | "当前正在进行维护，稍后手动处理" |
| 诊断有误 | "实际原因是电源问题，非温度问题" |
| 已有方案 | "已安排现场人员处理，无需远程操作" |
| 影响过大 | "该时段为收益高峰，暂不执行" |
| 需要核实 | "需要现场确认后再决定" |

## 3. 限电管理操作

### 3.1 如何使用 AI 限电建议功能

当电网要求限电或电价高峰时，AI 系统可帮助制定最优限电方案。

**获取 AI 限电建议步骤：**

1. 进入 **托管管理** → **智能限电管理**
2. 选择目标矿场
3. 点击 **获取24小时预测** 按钮

   - 建议限电时段
   - 预计节省电费
   - 算力损失评估

### 3.2 理解限电方案

系统提供三种限电策略供选择：

#### 效率优先策略 (Efficiency First)

**特点：**
- 优先关闭效率最低的矿机
- 最小化算力损失
- 适合需要保持产出的场景

**适用场景：**
- 短期限电（< 4小时）
- 比特币价格高位
- 对产出有明确要求

#### 平衡策略 (Balanced)

**特点：**
- 综合考虑效率和收益
- 均衡分配负载减少
- 适合大多数场景

**适用场景：**
- 中等时长限电（4-12小时）
- 常规限电需求
- 无特殊要求时的默认选择

#### 收益优先策略 (Revenue First)

**特点：**
- 优先考虑电费成本
- 关闭高功耗设备
- 最大化成本节省

**适用场景：**
- 长期限电（> 12小时）
- 电价高峰期
- 成本控制为主要目标

**策略对比表：**

| 策略 | 算力保留 | 成本节省 | 设备负担 | 推荐场景 |
|-----|---------|---------|---------|---------|
| 效率优先 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | 均匀 | 短期、高币价 |
| 平衡 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 均匀 | 日常限电 |
| 收益优先 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 集中 | 长期、高电价 |

### 3.3 执行限电操作的步骤

**步骤 1：设置限电参数**

1. 在限电管理页面选择策略
2. 设置目标功率减少量（kW）或关机比例（%）
3. 设置限电时长

**步骤 2：预览限电方案**

1. 点击 **预览方案**
2. 查看将被关闭的矿机列表
3. 确认影响评估

**步骤 3：执行限电**

1. 确认方案无误后，点击 **执行限电**
2. 系统显示确认对话框，再次确认
3. 输入您的操作密码（如果需要）
4. 等待执行完成

**步骤 4：监控限电状态**

执行后，可在仪表板监控：
- 已关闭矿机数量
- 实际节省功率
- 剩余限电时间

### 3.4 恢复矿机操作

#### 计划恢复（自动）

如果在限电时设置了自动恢复，系统将在指定时间自动开启矿机。

#### 手动恢复

1. 进入 **限电管理** 页面
2. 查看 **限电中** 的矿机列表
3. 选择要恢复的矿机（或全选）
4. 点击 **恢复运行**

#### 紧急恢复

遇到紧急情况需要立即恢复所有矿机：

1. 点击页面顶部的 **紧急恢复** 按钮（红色）
2. 确认紧急恢复操作
3. 系统将立即开启所有限电矿机

> ⚠️ **警告**：紧急恢复将取消所有计划的限电任务，请谨慎使用。

## 4. 告警诊断与工单

### 4.1 使用 AI 诊断告警

当系统产生告警时，可使用 AI 诊断功能快速分析根因。

**诊断步骤：**

1. 在告警列表中找到目标告警
2. 点击 **AI 诊断** 按钮
3. 等待分析完成（通常 3-10 秒）
4. 查看诊断报告

### 4.2 根据 AI 建议创建工单

当 AI 建议需要现场处理时，可快速创建工单。

**创建工单步骤：**

1. 在 AI 诊断报告或建议详情页点击 **创建工单**
2. 系统自动填充工单信息：
   - 标题
   - 问题描述
   - 相关设备
   - 优先级
3. 补充或修改信息（如需要）
4. 指派处理人员
5. 点击 **提交工单**

### 4.3 工单处理流程

工单从创建到关闭的完整流程：

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  待处理  │ →  │  处理中  │ →  │  待验证  │ →  │  已解决  │ →  │  已关闭  │
│  (Open)  │    │(Progress)│    │(Verify) │    │(Resolved)│    │ (Closed)│
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
```

**各状态说明：**

| 状态 | 操作人 | 说明 |
|-----|-------|------|
| 待处理 | 系统/创建者 | 工单已创建，等待分配或开始处理 |
| 处理中 | 处理人员 | 正在处理中，可更新处理进度 |
| 待验证 | 处理人员 | 处理完成，等待创建者验证 |
| 已解决 | 创建者 | 确认问题已解决 |
| 已关闭 | 系统 | 工单归档 |

**处理工单步骤：**

1. 进入 **技术支持** → **工单管理**
2. 找到待处理的工单
3. 点击 **开始处理**
4. 记录处理过程和结果
5. 完成后点击 **提交验证**

### 5.3 安全边界说明

系统设置了多层安全边界，确保操作安全：

#### 操作安全边界

| 边界类型 | 说明 | 默认值 |
|---------|------|-------|
| 单次最大设备数 | 单次操作影响的最大设备数 | 100 台 |
| 每日操作次数 | 每日可执行的操作总数上限 | 500 次 |
| 功率影响上限 | 单次操作最大功率变化 | 5000 kW |
| 连续操作间隔 | 对同一设备的最小操作间隔 | 5 分钟 |

#### 权限安全边界

- **最小权限原则**：用户仅能执行其角色允许的操作
- **二次确认**：高风险操作需要二次确认
- **操作审计**：所有操作可追溯
- **会话超时**：闲置 30 分钟自动登出

#### 系统安全边界

- **紧急停止**：任何时候可执行紧急停止
- **熔断机制**：异常操作自动停止
- **回滚能力**：支持操作回滚（部分操作）

> 🔒 **安全提示**：如需修改安全边界设置，请联系系统管理员并说明业务需求。

## 6. 故障排除

### 6.1 常见问题与解决方案

#### 问题 1：AI 建议长时间处于"执行中"状态

**可能原因：**
- 网络通信问题
- 目标设备离线
- 命令执行超时

**解决步骤：**
1. 检查目标设备的在线状态
2. 查看命令执行日志
3. 如果设备离线，等待设备恢复后重试
4. 如果持续超时，联系技术支持

#### 问题 2：自动审批规则不生效

**可能原因：**
- 规则未启用
- 条件不匹配
- 时间窗口外

**解决步骤：**
1. 检查规则状态是否为"已启用"
2. 确认当前时间在规则生效时间内
3. 验证建议的风险等级是否满足规则条件
4. 检查其他条件（设备数、操作类型等）

#### 问题 3：无法审批 AI 建议

**可能原因：**
- 权限不足
- 建议已过期
- 建议已被其他人处理

**解决步骤：**
1. 确认您的用户角色有审批权限
2. 检查建议的过期时间
3. 刷新页面确认建议的最新状态
4. 如果权限不足，联系管理员

#### 问题 4：限电操作执行失败

**可能原因：**
- 部分设备通信失败
- 目标功率设置不合理
- 矿机正在执行其他操作

**解决步骤：**
1. 查看执行报告，确认失败的设备
2. 检查失败设备的网络连接
3. 对失败设备单独重试
4. 如果问题持续，手动执行或联系技术支持

#### 问题 5：审计日志查询慢

**可能原因：**
- 查询时间范围过大
- 系统负载高

**解决步骤：**
1. 缩小查询时间范围
2. 添加更多筛选条件
3. 避开系统高峰期查询
4. 对于大量数据导出，使用 API 接口
