# Bitcoin独立数据分析系统

## 系统概述

已成功创建一个完全独立的Bitcoin数据分析系统，专门用于实时数据采集、技术分析和自动报告生成。系统与主应用分离，具备以下核心功能：

### 核心特性
- **实时数据收集**: 每15分钟自动采集BTC价格、网络算力、恐惧贪婪指数等数据
- **技术指标计算**: RSI、MACD、布林带、移动平均线等专业技术分析
- **自动报告生成**: 每日早晚8点生成分析报告和投资建议
- **独立Web仪表盘**: 实时数据可视化和报告查看界面
- **专用数据库表**: 独立的分析数据存储，不影响主应用

## 系统架构

### 数据收集引擎 (`analytics_engine.py`)
- **多源数据集成**: CoinGecko、Blockchain.info、恐惧贪婪指数API
- **智能调度**: 使用schedule库实现定时任务
- **数据验证**: 自动验证数据完整性和准确性
- **容错处理**: API失败时自动重试和降级处理

### 技术分析模块
- **价格趋势分析**: SMA20/50、EMA12/26移动平均线
- **动量指标**: RSI14、MACD信号线
- **波动性分析**: 布林带、30日波动率
- **市场情绪**: 恐惧贪婪指数解读

### 自动报告系统
- **日报生成**: 包含市场摘要、关键发现、投资建议
- **风险评估**: 基于多因子模型的风险评分
- **置信度评估**: 数据质量和分析可靠性评分
- **业务建议**: 针对挖矿投资的具体操作建议

### Web仪表盘 (`analytics_dashboard.py`)
- **实时数据展示**: 当前BTC价格、网络算力、技术指标
- **价格图表**: 24小时价格趋势可视化
- **分析报告**: 最新分析结果和建议展示
- **风险监控**: 实时风险评估和预警

## 数据库设计

### 专用分析表结构

#### market_analytics
```sql
- recorded_at: 记录时间
- btc_price: BTC价格
- btc_market_cap: 市值
- btc_volume_24h: 24小时交易量
- network_hashrate: 网络算力
- network_difficulty: 网络难度
- fear_greed_index: 恐惧贪婪指数
- price_change_1h/24h/7d: 价格变化
```

#### technical_indicators
```sql
- recorded_at: 计算时间
- sma_20/50: 简单移动平均线
- ema_12/26: 指数移动平均线
- rsi_14: 相对强弱指数
- macd: MACD指标
- bollinger_upper/lower: 布林带
- volatility_30d: 30日波动率
```

#### analysis_reports
```sql
- report_type: 报告类型
- generated_at: 生成时间
- title: 报告标题
- summary: 分析摘要
- key_findings: 关键发现(JSON)
- recommendations: 投资建议(JSON)
- risk_assessment: 风险评估(JSON)
- confidence_score: 置信度评分
```

```python

engine = AnalyticsEngine()
engine.collect_and_analyze()
engine.generate_daily_report()
```
