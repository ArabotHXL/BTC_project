# 网络数据收集与分析系统实施方案
## 设计目标与商业价值

### 1. 系统概述

我已为您实现了一个完整的BTC网络数据历史记录和分析系统，该系统能够：

**自动数据收集**
- 每次用户进行挖矿计算时自动记录网络快照
- 存储BTC价格、网络难度、全网算力、区块奖励
- 记录时间戳、数据来源、API响应时间

**智能数据分析**
- 价格趋势分析：7天/30天价格波动统计
- 难度调整分析：网络难度变化趋势
- 算力变化监控：全网算力增减分析
- 收益预测模型：基于历史数据的盈利预测

### 2. 数据库设计

新增 `network_snapshots` 表结构：
```sql
- id: 主键
- recorded_at: 记录时间（带索引）
- btc_price: BTC价格
- network_difficulty: 网络难度（T）
- network_hashrate: 网络算力（EH/s）
- block_reward: 区块奖励
- price_source: 价格数据源（coingecko）
- data_source: 网络数据源（blockchain.info）
- is_valid: 数据有效性标记
- api_response_time: API响应时间
```

### 3. 数据收集触发机制

**自动收集**
- 用户使用"实时数据"进行挖矿计算时自动记录
- 确保只有成功的计算才会记录数据
- 避免重复记录，减少存储开销

**手动收集**
- Owner/Admin角色可手动触发数据快照
- 用于系统维护或特殊时间点的数据记录

### 4. 数据分析功能

#### A. 价格趋势分析
**功能描述**：
- 分析BTC价格在指定时间段内的变化
- 计算价格变化率、最高价、最低价、平均价
- 生成可视化价格走势图

**商业价值**：
- 帮助客户了解市场趋势
- 为投资决策提供数据支持
- 评估最佳买入/卖出时机

#### B. 难度调整分析
**功能描述**：
- 监控网络难度调整周期
- 分析难度变化对挖矿收益的影响
- 预测未来难度调整趋势

**商业价值**：
- 预判挖矿竞争激烈程度
- 优化矿场运营策略
- 评估长期挖矿投资价值

#### C. 算力变化监控
**功能描述**：
- 追踪全网算力增减变化
- 分析算力波动对个人收益的影响
- 监控矿工进入/退出市场趋势

**商业价值**：
- 评估市场竞争环境
- 判断矿机升级时机
- 制定合理的扩容计划

#### D. 历史收益预测
**功能描述**：
- 基于历史数据模拟不同矿机的收益表现
- 分析电费成本对收益的影响
- 计算盈利率和投资回报周期

**商业价值**：
- 为客户提供数据驱动的投资建议
- 优化矿机配置和电费合同
- 降低投资风险

### 7. 访问权限控制

**权限分级**：
- Owner: 完整访问权限，可手动记录快照
- Admin: 查看所有分析功能
- Mining_Site: 查看基础分析功能
- Guest: 无权限访问

### 8. 路由结构

**页面路由**：
- `/network/history` - 网络历史数据分析页面

**API路由**：
- `/api/network/stats` - 网络统计概览
- `/api/network/price-trend` - 价格趋势数据
- `/api/network/difficulty-trend` - 难度趋势数据
- `/api/network/hashrate-analysis` - 算力分析数据
- `/api/network/profitability-forecast` - 收益预测
- `/api/network/record-snapshot` - 手动记录快照
