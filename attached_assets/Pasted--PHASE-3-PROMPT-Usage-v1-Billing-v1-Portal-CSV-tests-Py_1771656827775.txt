B) Usage Service（必须新增）
=====================
新增 services/usage_service.py：
- generate_usage(org_id, site_id, period_start, period_end, tenant_id=None):
  - 选择 miners：
    - 若 tenant_id 提供：仅该 tenant
    - 否则：org 下全部 tenant（operator 用）
  - 估算功耗优先级：
    1) 若存在 telemetry（例如 miner_telemetry_history / miner_telemetry_live 里有 watts + timestamp）：按采样积分估算 kWh
    2) 若无：使用 miners.nominal_watts * online_time
       - online_time：如果无状态历史，则假设 100% 在线并在 evidence_json 标明
  - 输出聚合到 tenant_id 的 kwh_estimated/avg_kw_estimated
  - evidence_json 写入：方法、采样点、缺失率、假设

注意：必须能在“无 telemetry 表”的情况下工作（fallback nominal_watts）。

=====================
C) Billing Service（必须新增）
=====================
新增 services/billing_service.py：
- generate_invoice(contract_id, period_start, period_end):
  - 读取 contract + tariff（Phase3 只支持 flat_rate：tariff.params_json.flat_rate）
  - 找到/生成 usage_record（该 tenant/site/period）
  - line_items：
    - electricity_cost = kwh_estimated * flat_rate
    - hosting_fee：
      - per_kw: avg_kw_estimated * rate_per_kw * duration（或按月固定）——选择最简单且写清楚口径
      - per_miner: miner_count * rate_per_miner
      - flat: fixed
    - curtailment_credit_split（可选 v1 简化）：
      - period 内若有 hi_curtailment_results（site/tenant 匹配），用 actual_json.saved_kwh * flat_rate * split_pct
      - 没有则 0
  - subtotal/total = sum(line_items)
  - evidence_json 引用 usage_record_id 与 curtailment_result_ids
  - 生成 hi_invoices(status='draft')

并实现 export_invoice_csv(invoice_id)：
- 输出列：item_type, description, amount, evidence_ref
