1) P0 修复：tenant_admin 掉级（必须）
========================
现状 bug：session['hi_tenant_id'] 已存在时跳过 membership 查询，g.hi_role 回落为 ROLE_TO_HI_ROLE['client']=tenant_viewer。
目标：每次请求都必须查 membership，用 membership.member_role 强制覆盖 g.hi_role（tenant_admin/tenant_viewer）。

在 init_hi_tenant() 的 TENANT 分支里实现以下逻辑（保持你已有的 403/空集保护不变）：

1. 获取 user_id（按现有实现：session 或 current_user）
2. candidate_tenant_id = session.get('hi_tenant_id')
3. membership = None
   - 若 candidate_tenant_id 存在：
       membership = HiTenantMembership.query.filter_by(user_id=user_id, tenant_id=candidate_tenant_id).first()
   - 若 membership 不存在：
       membership = HiTenantMembership.query.filter_by(user_id=user_id).order_by(
           HiTenantMembership.is_default.desc(),
           HiTenantMembership.tenant_id.asc()
       ).first()
4. 若 membership 存在：
   - tenant = HiTenant.query.get(membership.tenant_id)（或 db.session.get）
   - 若 tenant 存在且 tenant.status=='active'：
       g.hi_tenant_id = tenant.id
       g.hi_org_id = tenant.org_id
       g.hi_role = membership.member_role   # 关键：强制覆盖，防掉级
       session['hi_tenant_id'] = tenant.id  # 统一会话 tenant
     否则：
       g.hi_tenant_id = None
5. 若 membership 不存在：
   g.hi_tenant_id = None

注意：
- 不允许仅在 session 没有 tenant_id 时查 membership；必须每次都查
- 角色覆盖必须发生在每次请求（只要是 TENANT 用户）
- 若用户无 membership，后续由你现有的 hi_require_auth 装饰器返回 403 HI_TENANT_BINDING_REQUIRED
