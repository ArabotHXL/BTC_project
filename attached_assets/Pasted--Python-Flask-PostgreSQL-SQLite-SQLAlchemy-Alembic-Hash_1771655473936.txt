6) Curtailment 逻辑（必须实现 v1，可解释、可执行）
========================
实现一个 v1 策略引擎（services/curtailment_engine.py）：
输入：
- 电价（flat 或实时值）、目标：save_cost/keep_profit/thermal_protect
- 约束：min_online_pct、protected_groups、max_offline_groups
- 分组优先级：group.priority（数值越小越先关 or 你定义清楚并写入 README）
输出：
- actions：按 group 或 miner 生成 command（默认 power_off；可选 throttle）
- expected：
  - expected_kwh_saved（按 watts * duration）
  - expected_revenue_lost（若仓库已有 ROI/hashprice 计算则复用；否则用简化：hashrate * btc_price proxy，并在 README 标明简化口径）
执行：
- 将 actions 写 curtailment_actions，并推入现有 command_queue/outbox（若不存在则实现简单 command_queue 表 + worker stub）
验证：
- before_snapshot：执行前站点/tenant 总功耗与算力
- after_snapshot：执行后（延迟 N 秒可配置），计算 actual delta，写 curtailment_results

========================
7) 用电估算与分摊（必须实现 v1）
========================
usage generator（services/usage_service.py）：
- 按 period_start/end，聚合每个 miner 的功耗：
  - 优先：telemetry watts（若有）
  - 其次：miner.nominal_watts * online_time（若无遥测功耗）
- 汇总到 tenant_id + site_id 输出 kwh_estimated / avg_kw_estimated
- evidence_json 写入：采样点、缺失率、估算方法

invoice generator（services/billing_service.py）：
- 从 contract 取 tariff 与 hosting_fee 参数
- 计算 line_items：
  - electricity_cost = kwh_estimated * rate（flat 简化；TOU 可先不做或做最小）
  - hosting_fee（按 per_kw/per_miner/flat）
  - curtailment_credit_split（若 period 内有 curtailment_results，可先做 0 或做简单：saved_kwh * rate * split_pct）
- 输出 invoice（draft）+ export.csv
