1) 修复 tenant_admin 角色稳定性（P0 核心 bug）
========================
问题：当前逻辑在 session['hi_tenant_id'] 已存在时，会跳过 membership 查询，导致 g.hi_role 回落到 ROLE_TO_HI_ROLE['client']=tenant_viewer，而不是保持 membership.member_role=tenant_admin。

要求（必须做到）：
- TENANT 用户每个请求都以 membership 为准决定 g.hi_role
- 即使 session 已有 hi_tenant_id，也必须查询 membership 并覆盖 g.hi_role
- 如果 session 里的 hi_tenant_id 不属于该 user 的 membership，要自动回退到 default membership，并重写 session['hi_tenant_id']

在 common/hi_tenant.py 的 TENANT 分支里，按下面“最小改动”重构（保持你已有的 403/空集保护不变）：

A. 先拿 candidate_tenant_id：
   candidate_tenant_id = session.get('hi_tenant_id')  # 可能存在
B. 查 membership（必须）：
   membership = None
   if candidate_tenant_id:
       membership = HiTenantMembership.query.filter_by(
           user_id=user_id, tenant_id=candidate_tenant_id
       ).first()

   if not membership:
       # 回退到默认 membership（is_default desc, tenant_id asc）
       membership = HiTenantMembership.query.filter_by(user_id=user_id).order_by(
           HiTenantMembership.is_default.desc(),
           HiTenantMembership.tenant_id.asc()
       ).first()

C. 若 membership 存在：
   tenant = db.session.get(HiTenant, membership.tenant_id)
   if tenant and tenant.status == 'active':
       g.hi_tenant_id = tenant.id
       g.hi_org_id = tenant.org_id
       # 关键：强制覆盖角色（确保 tenant_admin 不掉级）
       if membership.member_role in TENANT_ROLES:
           g.hi_role = membership.member_role
       session['hi_tenant_id'] = tenant.id
   else:
       # tenant 不存在或非 active，当作无绑定
       g.hi_tenant_id = None
D. 若 membership 不存在：
   g.hi_tenant_id = None
   # 让你已有的 hi_require_auth / hi_require_tenant_or_operator 走 403 HI_TENANT_BINDING_REQUIRED

注意：
- 不要只在 “session 没 tenant_id” 才查 membership
- 这是 P0 Gate 的根因修复
