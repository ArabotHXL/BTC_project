下面给你一套可直接落地到你现有 HashInsight 的产品蓝图 + 策略模板 + 指标/告警 + 回测与执行，都以“信息支持，不构成投资建议”为前提。

一、平台要解决的三件事

知道卖多少：覆盖近期开销与现金流（OPEX/CapEx/贷款/利息/托管费/电费）。

知道什么时候卖：把“行情/矿工指标/技术信号/风险”合成明确触发器。

怎么卖更好：分批/限价/TWAP/VWAP/OTC，尽量降滑点与税负，必要时用对冲锁价。

二、核心输入（一次建模，日常只改少数参数）

库存与成本：当前 BTC 库存、分批入账成本（支持 FIFO/LIFO/Specific ID）、未实现盈亏。

现金需求：未来 7/30/90 天 OPEX、到期账单、最低现金余额（Runway）。

风险偏好：保守/中性/激进；单日最大可卖比例；允许对冲（是/否）。

市场与链上数据：现货价、波动率(ATR/Realized Vol)、资金费&基差、难度/Hashprice、Puell Multiple、SOPR/MVRV、矿工流向(MPI)。

执行场所与限额：可用交易所、额度、最小成交量、最差成交价（Worst-Case Limit）。

三、策略模板（直接给客户可选）

A. OPEX 覆盖规则（现金流优先）

目标：确保未来 30/60 天现金安全。

逻辑：若 现金+稳定币 < 未来OPEX，则卖出 缺口金额 / 现价 的 BTC（上限按“最大日度卖出%”）。

优点：简单稳健；缺点：不择时，仅保生存。

B. 分层止盈 + 尾仓追踪止损（最常用）

阶梯价：以成本基准或52周百分位定 3–5 个卖出层级（例如：1.2×成本、1.5×、1.8×、2.2×）。

配额：每层卖 10–25%，最后留 20–40% 尾仓。

尾仓：设追踪止损，如 最高收盘价 × (1 − 8%) 触发清仓/减仓。

可加超买过滤：RSI>70 或 20日新高才触发对应层级。

C. Hashprice/矿工周期驱动

指标：Hashprice 分位数、Puell Multiple、Hash Ribbons（矿工投降/恢复）。

规则示例：

当 Hashprice > 历史70%分位 且 Puell > 1.8 → 卖 15%；

当出现 Hash Ribbons Recovery 且价格创新高 → 再卖 10%。

作用：把“矿工景气度”纳入择时。

D. 基差&资金费驱动（带对冲选项）

若 年化基差 > X%（现货强于期货）且你想锁价而不卖币：

做空等额永续/期货（对冲暴露），持币赚基差；

到期/条件满足时再平仓或交割。

平台给出教育式说明（非建议），并标注风险（强平/资金费方向切换）。

E. 波动率/风控触发

价格从30日高点回撤 ≥ 10% 或 ATR 突增到一年80%分位 → 卖 10–20%。

大幅回撤时自动切换到“保守模式”，提高尾仓止损灵敏度。

四、指标与触发（可配置）

技术面：MA(50/200) 趋势、布林/ATR 通道突破、RSI 拐点/背离。

链上/矿工：Puell、SOPR、MVRV、MPI、矿工→交易所流量。

微观结构：买卖盘深度、点差、资金费/基差、未平仓量突增。

资金与税：距税期剩余天数、未实现税负/税盾、现金覆盖天数。

每个触发器都输出**“建议动作 + 可信度/冲突提示”，并给人话解释**。

五、执行方案（降滑点）

分批限价（Ladder LMT）：分 3–6 档挂单，随价上移。

TWAP/VWAP：在 30–120 分钟内均匀完成，避开新闻波动窗口。

场景切换：小额走现货，大额走 OTC/Block Trade；流动性差时降速。

六、回测与评估（把“感觉”变成“证据”）

回测对象：把“挖矿产出曲线 + 决策规则”合成现金流，再与“简单 DCA/只囤不卖/一次性卖”的基准对比。

指标：累计收益、最大回撤、卡玛比/夏普、命中率、滑点、成交成本、收益波动。

可视化：

库存 & 成本基线 vs 价格；

触发点标记（卖出/对冲）；

现金余额与 OPEX 覆盖天数。

七、界面草图（直接给你前端落地）

Treasury 概览：库存BTC、成本、未实现盈亏、现金覆盖天数、下月应付。

Sell Plan 向导：目标（保现金/锁收益/进取）→ 选择模板 A/B/C/D → 自动生成卖出梯级 & 止损。

信号面板：技术/矿工/链上/微观各一列，绿黄红灯＋最近触发说明。

执行面板：一键生成挂单/TWAP 指南（不托管 API 的“手动执行清单”模式，或可选接交易所 API）。

告警：价格穿越阈值、追踪止损、生存金不足、基差异常、Hashprice 触顶等（邮件/Telegram/Discord Webhook）。

八、合规与会计（必备）

披露：全站“教育用途，不构成投资建议”。

税务：支持 FIFO/LIFO/Specific ID 成本法、已实现/未实现收益报表、导出对账。

权限：不保存交易所密钥或仅只读；加密存储、可撤销。

地域：披露衍生品风险与地域限制。

九、你现在就能加的三块 MVP（不动你现有批量导入）

OPEX 覆盖规则 + 分层止盈 + 尾仓追踪（三合一基础策略）。

信号解释引擎：把触发原因用一句话生成（例：“价破 52W 高且 RSI>70，卖 15% 第2档”）。

回测器：读取历史价、模拟从你产出表流入的 BTC 库存，输出 P&L 与触发点。

示例触发（可直接硬编码为默认策略）

层1：Price ≥ 1.2 × 加权成本 → 卖 15%

层2：Price ≥ 1.5 × 加权成本 且 RSI14>65 → 卖 15%

层3：Price ≥ 2.0 × 加权成本 或 52W 百分位 ≥ 90% → 卖 20%

尾仓追踪：Close ≤ 最高收盘 × (1 − 8%) → 卖 50% 尾仓

风控：30D 回撤 ≥ 12% 且 ATR 占价比 ≥ 2.5% → 进入保守模式（所有卖出档位各+5%配额）

现金缺口：现金 < 30D OPEX → 立即卖 缺口/现价（不超“单日上限”）
