# Analytics Dashboard â€” Quickstart

## 1) Create and activate a virtual environment
```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate
```

## 2) Install dependencies
```bash
pip install -r requirements.suggested.txt
```

## 3) Configure environment
Copy `.env.example` to `.env` and edit values:
```bash
cp .env.example .env
```

## 4) Initialize the database (dev)
If using the default SQLite URL from `.env`, no DB server is required. The app will create tables on first run (SQLAlchemy).

## 5) Run the backend
```bash
# From the extracted folder:
cd /mnt/data/analytics_dashboard_extracted/analytics_dashboard_files/backend
python main.py
```
The service will start on http://127.0.0.1:5000 and expose:
- UI: `/analytics_dashboard` or `/analytics/main`
- Health: `/health`, `/ready`, `/alive`
- API: `/api/analytics/*`, `/api/btc-price`, `/api/calculate`, etc.

## 6) Example request
```bash
curl http://127.0.0.1:5000/api/btc-price
```

> Tip: Set `ENABLE_BACKGROUND_SERVICES=1` in `.env` to enable scheduled refresh jobs (15-min cadence).
