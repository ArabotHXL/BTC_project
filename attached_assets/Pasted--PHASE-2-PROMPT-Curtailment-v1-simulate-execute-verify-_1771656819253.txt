=====================
B) Curtailment Engine（必须新增）
=====================
新增 services/curtailment_engine.py，实现 v1：
输入（来自 curtailment_plans.inputs_json）：
- duration_minutes（默认 60）
- power_price（可选）
- min_online_pct（默认 0.6）
- protected_group_ids（默认 []）
- max_offline_groups（默认 null）
策略：
- 选择范围：
  - site_wide：该 site 下所有 groups
  - tenant_only：该 tenant 下 groups
- 排序：
  - 按 group.priority（数值越小越先关）或相反，但必须在 README 写清楚
- 生成 actions：
  - 逐个 group 生成 power_off，直到满足 min_online_pct 或达到 max_offline_groups
- expected_json：
  - expected_kw_saved = sum(group_selected_miners_nominal_watts)/1000
  - expected_kwh_saved = expected_kw_saved * duration_hours
  - expected_revenue_lost：若仓库已有 ROI/hashprice 模块则复用；否则用简化模型：
      revenue_proxy = online_hashrate_th * hashprice_usd_per_th_day * duration_days
    其中 hashprice_usd_per_th_day 可从 inputs_json 传入（默认 0 也可以）并在 README 标明简化。
输出：
- actions[]（目标、命令、payload）
- expected_json

Snapshot（before/after）：
- 汇总站点/tenant 的 miners 数量、online 数、nominal_watts 总和、（若有）telemetry_watts 总和
写入 results.actual_json：实际降载 kw/kwh（用 nominal 或 telemetry，按可得数据）

=====================
C) API（Phase2）
=====================
新增/补齐以下端点（tenant 隔离必须正确）：
- POST /api/curtailment/plans
  body: {site_id, tenant_scope, tenant_id?, objective, inputs_json}
  returns: plan
- POST /api/curtailment/plans/<id>/simulate
  returns: {expected_json, actions_preview}
- POST /api/curtailment/plans/<id>/execute
  行为：
    1) 生成 actions（如果 expected_json 为空可先 simulate）
    2) 写 hi_curtailment_actions（status=queued）
    3) 推入命令队列（现有或 hi_command_queue）
    4) plan.status=executing
    5) 写 hi_audit_log(action_type='CURTAILMENT_EXECUTE')
  returns: {request_id, actions_count}
- POST /api/curtailment/plans/<id>/verify
  行为：
    1) 计算 before/after snapshot
    2) 写 hi_curtailment_results
    3) plan.status=completed
    4) 写 hi_audit_log(action_type='CURTAILMENT_VERIFY')
- GET /api/curtailment/plans?site_id=&tenant_id=
- GET /api/curtailment/plans/<id>/report
  returns: {plan, actions, results}

命令状态：
- GET /api/commands/<request_id>（如果已有命令查询 API 则复用，否则从 hi_command_queue 查询）
